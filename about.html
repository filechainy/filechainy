About


Our objective is create a decentralized storage system with a fair reward mechanism.

The basic system model is that each server must be capable to check if the others servers hosting the same files. Each node also can compare if the registers of the files the same in the others nodes (that is, a database only with URLs)

The design principle aim is that even the lowest device in computational power should be able to run a node.

For this be possible a node must not perform all tasks and not many. But there must be different types of nodes, for example, some that just send files, others that host or validate.

One aspect that can make nodes faster and simpler is by default saving and sending files in chunks. So there's no rework to split or assemble a file every time it's uploaded.

Lightweight nodes can send and download files via http GET request.

Decentralized network


A decentralized network must take into account:

1) The ease with which a node can participate in the network or download all of its content

2) Save pieces of the same file on different servers
