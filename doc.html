Documentation

Index.php


In index.php you can insert new items through the GET variables hash and contents

That is, if you access 'index.php?hash=test.txt&contents=123' you write a file called 'test.txt' with the contents '123' inside the 'hash' directory

You also can use the variable user for insert your payment method

Remember the system works with base64 encode and text file, not binary files

You get the path of a file inside 'hash' assigning to a variable id (e.g: index.php?id=1). You can also use userid, fileid and urlid to get the path of another directory

Receive.php


Receive the GET variables hash, contents and user. But shows a blank page if if the variables are not assigned

Send.php, multiple_send.php and multiple_send_list.php


These scripts act listing the files inside 'hash' folder and sending its contents to one or more urls

Convert.php


Split the files of 'file' folder in chunks of 8 Kb with a SHA1 name and save in 'hash' folder encoded in base64. A key will be created in 'key' folder with the order that the chunks of the file must be read

View.php


Paste the pieces of files present in a key (restore a file)

Other scripts

upload.php : Upload files to the 'files' directory

filelist.php : Show the files uploaded (as text)

gallery.php : Show the files uploaded (as image)

urls.php : Enter manually a URL

paste.php : Simple paste tool

groups.php : Send a message to a group

zip_hash.php : Zip the contents of the folder 'hash'

files.php : Get the path of the files in a directory by GET variable id


Default directories

files : Common files (binary files)

hash : The chunk of files with SHA1 name encoded in base64

user : Store the user that send a chunk of a file

url : URLs and servers available


Java

In the 'java' folder you will find several basic tools for you build your own node validator or searcher. But the core is 'CheckUrls'

CheckUrls.java

This program read the text file 'urls.txt'and check each URL against all others for verify if they host the same files. If so, a small GET request will be performed to a log server. The files that will be checked must be in the 'hash' folder.
