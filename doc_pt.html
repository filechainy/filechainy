Documentação

Index.php


No index.php você pode inserir novos arquivos através da variável GET hash e contents

Ou seja, se você acessar 'index.php?hash=test.txt&contents=123' você irá escrever um arquivo chamado 'test.txt' que terá como conteúdo o texto '123' sendo guardado dentro da pasta 'hash'

Você também pode usar a variável user para inserir o seu endereço de pagamento. Ou seja, o endereço de pagamento associado a cada pedaço do arquivo

Lembre-se o sistema trabalha apenas com arquivos de texto e codificados em base64

Você obtém o caminho de um arquivo da pasta 'hash' iniciando a variável id (e.x: index.php?id=1). Você também pode usar userid, fileid ou urlid para obter o caminho de um arquivo de outro diretório

Receive.php


Recebe e escreve as variáveis GET hash, contents and user. Mostra uma página em branco se não for inicializada

Send.php, multiple_send.php and multiple_send_list.php


Esses scripts servem para enviar os arquivos dentro da pasta 'hash' para alguma URL (servidor) ou lista de URLs

Convert.php


Divide os arquivos da pasta 'file' em pedaços de 8 Kb com um nome SHA1 e os salva na pasta 'hash' codificados em base64. Uma chave será criada no diretório 'key' com a ordem correta cujo qual os arquivos devem ser reorganizados

View.php


Junta os pedaços de arquivos presentes em uma chave (restabelece o arquivo)

Outros scripts

upload.php : Envie arquivos para o diretório 'files'

filelist.php : Mostra os arquivos enviados (modo texto)

gallery.php : Mostra os arquivos enviados (como imagens)

urls.php : Adicione uma URL manualmente

paste.php : Ferramente simples para colagem

groups.php : Envie uma mensagem para um grupo

zip_hash.php : Compacta o conteúdo do diretório 'hash'

files.php : Obtém o caminho dos arquivos em um diretório através da variável GET id


Diretórios padrão

files : Arquivos comuns

hash : Pedaços de arquivos com um nome SHA1 e codificados em base64

user : Armazena o usuário que enviou o pedaço de um arquivo

url : URLs e servidores disponíveis

log : Diretório reservado para guardar validações realizadas pelo próprio servidor e por outros


Java

Dentro da pasta 'Java' você encontrará ferramentas básicas para construir o seu próprio validador. Mas, a aplicação central é a 'CheckUrls'

CheckUrls.java

Esse programa lê um arquivo de texto chamado 'urls.txt' e checa cada URL com as demais para verificar se elas hospedam os mesmos arquivos. Em caso positivo, uma requisição será realizada em um servidor de log. Os arquivos que serão comparados devem estar dentro do diretório 'hash' no computador ou servidor do próprio usuário
